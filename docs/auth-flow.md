# SupplyHub 인증 흐름

이 문서는 로그인, 회사 온보딩, 회사 권한 확장 흐름을 정의하는 기준 문서다.
역할별 권한과 제약 자체는 [roles.md](./roles.md)를 따른다.

## 흐름 구조

- `로그인`
- `회사 온보딩`
- `회사 권한 확장`

설명:

- `로그인`은 외부 인증으로 사용자를 식별하는 단계다.
- `회사 온보딩`은 로그인한 사용자를 회사와 연결하는 단계다.
- `회사 권한 확장`은 온보딩 완료 후 운영 권한을 확장하는 단계다.

## 로그인 시작

1. 사용자가 로그인 페이지에 진입한다.
2. 서버는 등록된 `ClientRegistration` 목록으로 로그인 가능한 provider를 표시한다.
3. 사용자는 `/oauth2/authorization/{registrationId}`로 외부 인증을 시작한다.

## 외부 인증 완료

- 인증 성공 후 서버는 외부 계정 정보를 정규화한다.
- 주요 속성은 아래를 포함한다.
- `registrationId`
- `providerUserId`
- `email`
- `resolvedName`
- `phone`
- `emailVerified`

## 로그인 후 분기

- `PLATFORM_ADMIN` 계정이면 플랫폼 운영 계정으로 인증된다.
- 이미 내부 회사 사용자 계정에 연결된 일반 사용자면 회사 사용자 계정으로 인증된다.
- 아직 회사에 연결되지 않은 일반 사용자면 회사 미연결 로그인 상태로 인증된다.
- 현재 구현에서는 위 세 경우 모두 로그인 성공 후 `/`로 리다이렉트한다.
- 이후 홈 화면에서 계정 상태에 따라 다음 이동 경로와 사용 가능한 액션이 갈린다.

## `PLATFORM_ADMIN` 로그인

- 인증 단계 자체는 일반 로그인과 같다.
- 인증 후 회사 온보딩으로 가지 않는다.
- 로그인 성공 후 `/`로 이동하고, 홈 화면에서 플랫폼 운영 영역으로 이어진다.

## 회사 미연결 로그인 사용자 상태

- 로그인은 성공했지만 회사에는 아직 연결되지 않은 상태다.
- 이 상태에서는 아래만 허용한다.
- 자기 인증 정보 확인
- 상품 조회
- 회사 온보딩 진행
- 로그아웃

## 회사 온보딩 분기

회사를 아직 연결하지 않은 로그인 사용자는 아래 둘 중 하나를 진행한다.

- `신규 회사 등록 요청`
- `기존 회사 참여 요청`

`최초 COMPANY_ADMIN 요청`은 온보딩이 아니라 온보딩 완료 후의 `회사 권한 확장` 단계다.

## 신규 회사 등록 요청

1. 로그인 사용자가 회사명과 회사 도메인을 입력해 `신규 회사 등록 요청`을 생성한다.
2. 승인 전에는 `Company`와 회사 연결 `User`를 만들지 않는다.
3. `PLATFORM_ADMIN`이 요청을 승인 또는 거절한다.
4. 승인 시 새 회사를 생성한다.
5. 로그인한 외부 인증 사용자에 대응하는 내부 사용자 계정을 생성하거나, 기존의 활성 미연결 사용자 계정을 재연결한다.
6. 그 내부 사용자 계정을 새 회사에 소속시킨다.
7. 현재 로그인에 사용한 외부 인증 정보를 그 내부 사용자 계정에 연결한다.
8. 기본 구매 역할 `CART_USER`를 부여한다.
9. 운영 역할은 부여하지 않는다.
10. 최종 상태는 `소속 완료`다.

## 기존 회사 참여 요청

1. 로그인 사용자가 기존 회사 참여를 선택한다.
2. 초대코드를 입력한다.
3. 대상 회사를 확인한다.
4. 즉시 회사 소속이 되지 않는다.
5. `회사 참여 요청`을 생성한다.
6. 현재 상태는 `회사 참여 승인 대기`가 된다.
7. `COMPANY_ADMIN`이 승인 또는 거절한다.
8. 승인 시 내부 회사 사용자 계정을 생성 또는 연결한다.
9. 현재 로그인 외부 인증 정보를 그 내부 회사 사용자 계정에 연결한다.
10. 기본 구매 역할 `CART_USER`를 부여하거나 유지한다.
11. 최종 상태는 `소속 완료`다.

추가 규칙:

- 초대코드 입력은 로그인 후에만 가능하다.
- 같은 사용자와 같은 회사 기준으로 승인 대기 참여 요청은 1건만 허용한다.
- 사용자는 자기 승인 대기 요청을 직접 취소할 수 있다.
- 취소 후와 거절 후에는 다시 요청할 수 있다.

## 승인 대기 상태

- `회사 참여 승인 대기` 상태의 사용자는 아직 그 회사 소속이 아니다.
- 이 상태에서는 아래만 허용한다.
- 자기 인증 정보 확인
- 상품 조회
- 요청 상태 확인
- 요청 취소
- 로그아웃

## 회사 참여 요청 승인

- `COMPANY_ADMIN`이 승인한다.
- 같은 회사에 같은 이메일의 활성 내부 사용자 계정이 이미 있으면 새 계정을 만들지 않고 기존 계정에 연결한다.
- 같은 회사에 같은 이메일의 비활성 내부 사용자 계정이 있으면 자동 활성화하지 않는다.
- 승인 시 현재 로그인 외부 인증 정보를 `UserIdentity`로 연결한다.
- 기존 활성 계정에 연결하는 경우 기존 구매 역할과 운영 역할 상태는 유지한다.

## 회사 참여 요청 거절

- `COMPANY_ADMIN`이 거절한다.
- 거절 사유는 필수 입력이다.
- 거절 후 사용자는 기본 온보딩 상태로 돌아간다.
- 내부 처리 메모는 사용자에게 노출하지 않는다.

## 회사 권한 확장

- 회사 온보딩 완료 후 별도 단계로 진행한다.
- 현재 범위의 핵심 절차는 `최초 COMPANY_ADMIN 요청`이다.
- `회사 생성 완료 트랜잭션을 최종 성공시킨 사용자` 1명만 요청할 수 있다.
- 요청은 `PLATFORM_ADMIN`이 승인 또는 거절한다.
- 승인 전 회사 생성자는 운영 역할 없이 동작한다.

## 로그아웃과 재로그인

- 온보딩 중 사용자가 로그아웃하면 진행 중 세션 상태는 종료된다.
- 승인 후 다시 로그인하면 연결된 회사 상태로 진입한다.
- 승인 전이나 거절 후에는 다시 온보딩 흐름으로 진입한다.

## 임시 인증 권한

- `ROLE_FIRST_LOGIN_COMPANY_SETUP`는 영속 역할이 아니다.
- 회사 연결 완료 전 임시 인증 상태에서만 사용한다.

## 현재 구현 기준 메모

- 신규 회사 등록은 `company_registration_requests` 기반 승인 흐름으로 구현돼 있다.
- 기존 회사 참여는 `company_join_requests` 기반 승인 흐름으로 구현돼 있다.
- 승인/거절 처리 시 `review_memo`, `rejection_reason`를 사용한다.
- 회사 탈퇴 후 active 상태를 유지한 채 회사 미연결 상태로 돌아갈 수 있다.
